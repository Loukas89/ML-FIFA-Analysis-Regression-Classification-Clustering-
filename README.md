# Machine Learning FIFA Analysis

Αυτό το project αποτελεί μια εργασία μηχανικής μάθησης, στην οποία εφαρμόζονται τρεις βασικοί τύποι ανάλυσης:

1. Παλινδρόμηση (Regression)
2. Ταξινόμηση (Classification)
3. Ομαδοποίηση (Clustering)

Το project χρησιμοποιεί datasets σχετικά με το ποδόσφαιρο και εφαρμόζει διαφορετικούς αλγόριθμους μηχανικής μάθησης με σκοπό την ανάλυση, την πρόβλεψη και την ομαδοποίηση δεδομένων.

Ο βασικός στόχος δεν είναι μόνο να εκπαιδευτούν μοντέλα, αλλά να παρουσιαστεί ολόκληρη η διαδικασία ενός machine learning project, από τη φόρτωση των δεδομένων μέχρι την αξιολόγηση και την ερμηνεία των αποτελεσμάτων.

---

## Regression Evaluation Metrics

Τα regression μοντέλα αξιολογήθηκαν με τα παρακάτω metrics:

```text
MAE
RMSE
R2 Score
```

### MAE

Το Mean Absolute Error δείχνει τη μέση απόκλιση ανάμεσα στις πραγματικές και τις προβλεπόμενες τιμές attendance.

### RMSE

Το Root Mean Squared Error μετρά επίσης το σφάλμα πρόβλεψης, αλλά δίνει μεγαλύτερη βαρύτητα στα μεγάλα λάθη.

### R2 Score

Το R2 Score δείχνει πόσο καλά το μοντέλο εξηγεί τη μεταβλητότητα της μεταβλητής στόχου. Όσο μεγαλύτερη είναι η τιμή, τόσο καλύτερη είναι η απόδοση του μοντέλου.

## Σύνοψη Regression

Η ανάλυση παλινδρόμησης έδειξε ότι μοντέλα όπως το Random Forest μπορούν να αποδώσουν καλά, επειδή μπορούν να εντοπίσουν πιο σύνθετα μοτίβα μέσα στα δεδομένα.

Η τελική αξιολόγηση βοήθησε στη σύγκριση των μοντέλων και στην επιλογή της πιο κατάλληλης προσέγγισης για την πρόβλεψη της προσέλευσης θεατών.

Το FIFA 23 players dataset ήταν η πιο κατάλληλη επιλογή για clustering, επειδή περιέχει πολλά αριθμητικά χαρακτηριστικά για κάθε παίκτη.

Για παράδειγμα, οι παίκτες μπορούν να ομαδοποιηθούν σε:

## Clustering Evaluation

Τα clustering αποτελέσματα αξιολογήθηκαν με:

```text
Silhouette Score
Inertia
Noise Points
Cluster Profile Analysis
PCA Visualisation
```

### Silhouette Score

Το Silhouette Score δείχνει πόσο καλά έχουν διαχωριστεί τα clusters. Όσο πιο υψηλή είναι η τιμή, τόσο καλύτερος θεωρείται συνήθως ο διαχωρισμός.

### Inertia

Το Inertia χρησιμοποιείται στο K-Means και δείχνει πόσο κοντά βρίσκονται τα σημεία στα κέντρα των clusters τους.

Χαμηλότερο inertia συνήθως σημαίνει πιο συμπαγή clusters, αλλά πρέπει να ερμηνεύεται προσεκτικά, επειδή μειώνεται φυσικά όταν αυξάνεται ο αριθμός των clusters.

### Noise Points

Τα noise points αφορούν κυρίως το DBSCAN. Είναι τα σημεία που ο αλγόριθμος δεν κατάφερε να εντάξει καθαρά σε κάποιο cluster.

### PCA Visualisation

Το PCA χρησιμοποιήθηκε για να μειώσει τα δεδομένα σε δύο διαστάσεις, ώστε τα clusters να μπορούν να απεικονιστούν σε ένα απλό 2D γράφημα.

Τα PCA components δεν αντιστοιχούν σε ένα μόνο αρχικό χαρακτηριστικό. Είναι συνδυασμοί πολλών χαρακτηριστικών μαζί.

## Σύνοψη Clustering

Μετά τη σύγκριση των clustering μοντέλων, το K-Means με `k=3` επιλέχθηκε ως η καλύτερη πρακτικά χρήσιμη λύση.

Αν και κάποιες παραλλαγές του DBSCAN εμφάνισαν υψηλότερο Silhouette Score, δημιούργησαν μόνο ένα βασικό cluster, κάτι που δεν ήταν ιδιαίτερα χρήσιμο για την ανάλυση.

Το K-Means με `k=3` δημιούργησε τρεις πιο ερμηνεύσιμες ομάδες παικτών:

1. Ποιοτικοί και τεχνικά δυνατοί παίκτες
2. Αμυντικοί και σωματικά δυνατοί παίκτες
3. Γρήγοροι παίκτες μέτριου επιπέδου

Αυτό έκανε τα αποτελέσματα πιο εύκολα στην κατανόηση και στην ερμηνεία.
